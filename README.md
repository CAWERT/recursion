Вариант 2.
Анализ алгоритма:

    Временная сложность: O(n), где n — показатель степени. На каждом шаге мы уменьшаем степень на 1.
    Пространственная сложность: O(n) из-за глубины рекурсии.


Преимущества и ограничения.
Преимущества:

    Простота реализации

    Легко понять принцип работы

    Работает для любых чисел (целых и дробных)

Ограничения:

    При больших значениях n может возникнуть переполнение стека

    Оптимизированная версия более эффективна для больших степеней

    Для отрицательных степеней требуется обработка деления
